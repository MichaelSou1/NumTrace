## 0. 新研究边界和复用原则

**Spec**

新的研究问题是：

> 在 chart/table 数值任务中，最终答案的因果路径是主要经过 localized visual numeric source，经过 textualized CoT number，还是二者在不同阶段分工？

这条线和当前 `Reading, Not Reasoning` 的关系：

- 当前 paper：主要问 emitted written CoT 是否 load-bearing。
- 新 research：估计 visual source、textualized number 和 final answer 之间的 path-specific causal effects。
- 当前 paper 的 random/two-pass battery 只能作为 diagnostic baseline 或 motivation。
- 新 research 的 headline evidence 必须来自 source-localized、answer-recomputable、same-target、in-place 或 four-cell path-specific 实验。

## 1. Core Design: 2x2 Path-Specific Conflict Audit

**Spec**

每个 valid attempt 围绕同一个 on-path numeric source 构造四格：

| visual source | emitted / forced CoT number | cell name | purpose |
|---|---|---|---|
| true | true | `V_true_T_true` | base sanity |
| false | false | `V_cf_T_cf` | coherence check |
| false | true | `V_cf_T_true` | visual source influence with text held fixed |
| true | false | `V_true_T_cf` | textualized number influence with visual source held fixed |

主变量：

- `v_true`：原始视觉源和原始 CoT 中的数值。
- `v_cf` / `v_inj`：同一 source 的 counterfactual value。
- `c_true`：原始 gold program / parsed expression 的最终答案。
- `c_cf`：把 source value 从 `v_true` 改成 `v_cf` 后重算的最终答案。
- `T_cf`：优先通过 in-place KV force 实现；不能 re-prompt。
- `V_cf`：优先通过 counterfactual rendering/editing 实现；不能只改 prompt text。

主读数：

- `E_visual_to_text = P(CoT_number = v_cf | V_cf) - P(CoT_number = v_cf | V_true)`。
- `E_text_to_answer = P(answer = c_cf | V_true, T_cf) - P(answer = c_cf | V_true, T_true)`。
- `E_visual_direct_to_answer = P(answer = c_cf | V_cf, T_true) - P(answer = c_cf | V_true, T_true)`。
- `conflict_winner = both_cf | visual_cf | text_cf | snap_true | other | non_identifiable_conflict`。

**Steps**

- [ ] 定义 `OnPathTarget`：
  `sample_id, target_id, token_span, v_true, v_cf, c_true, c_cf, single_token, backend, evidence`。
- [ ] 定义 `VisualSourceTarget`：
  `sample_id, source_id, source_type, source_region, source_surface, v_true, v_cf, c_true, c_cf, editable, renderer_backend, evidence`。
- [ ] 定义 `PathProbeRow`：
  `sample_id + target_id + path_cell` 必须可 join。
- [ ] 定义 `classify_readout(ans, c_true, c_cf)`：
  `snap -> follow -> other`。
- [ ] 定义 `classify_conflict(ans, c_true, c_source_cf, c_text_cf)`：
  `both_cf -> visual_cf -> text_cf -> snap_true -> other`；若 `c_source_cf == c_text_cf`，标为 `non_identifiable_conflict`。
- [ ] 每个 dataset backend 声明：
  source 可定位性、source 可编辑性、`c_cf` 可重算性、CoT token 可定位性。

**Acceptance**

- [ ] 每个进入主表的样本至少能填满 `V_true_T_true`、`V_true_T_cf`、`V_cf_T_true` 三格。
- [ ] `V_cf_T_cf` 是 coherence check，不能替代冲突格。
- [ ] `c_cf` 永远来自 source-level recompute，不是 injected operand 本身。
- [ ] `visual_direct` 和 `text_mediated` 两个 effect 可以在同一批样本上 paired 比较。

## 2. Data and Source-Family Selection

**Spec**

数据集不按名气选，而按是否支持 path-specific causal audit 选。每个候选任务先打五个分：

1. `source_localizable`：能否定位到 cell/bar/point/region/object set。
2. `source_editable`：能否生成无 artifact 的 counterfactual visual input。
3. `answer_recomputable`：改 source 后能否确定 `c_cf`。
4. `cot_token_locatable`：模型是否会写出可定位的 numeric intermediate。
5. `visual_realism`：真实感和外部效度。

推荐路线：

| tier | dataset/task | source type | role | risk |
|---|---|---|---|---|
| v1 | FinQA rendered table | table cell | gold-program clean substrate | 任务偏金融表格 |
| v1 | synthetic rendered table | table cell | renderer/cache spike | synthetic realism weak |
| v1.5 | TabMWP | table cell / expression operand | current headline table arithmetic | gold program weak，需要 parse-and-skip |
| v2 | PlotQA / DVQA / FigureQA / LEAF-QA | chart mark / data point | first non-table source family | chart editing/OCR/tick confounds |
| v2 | TAT-QA table-only | financial table cell | realistic financial table | text evidence mixed |
| v2 | HiTab / MultiHiertt | hierarchical table cell | hierarchy table source | alignment/rendering hard |
| v3 | MapQA / MapVerse | region color + legend | region/legend path | legend-color confounds |
| v3 | TallyQA / HowMany-QA / CLEVR / GQA | object set/count | non-printed-number source | real image editing hard |
| v3 | GeoQA / Geometry3K / MathVista / MathVerse / DynaMath | diagram relation | diagram numeric path | source edit hard |
| v3 | SPIQA / SciGraphQA / SciTabQA / SCITAT / SciNumQA | scientific figure/table | external validity | derivation often unavailable |

**Steps**

- [ ] 新增 `docs/snapshots/0628_dataset_selection.md`。
- [ ] v1 只选 source edit 最干净的 table/cell substrate，不从 ChartQA free-form 起步。
- [ ] 每扩一个数据集，先做 `n=30` source audit：
  source 是否可定位、counterfactual render 是否自然、模型是否写目标数字、`c_cf` 是否可重算。
- [ ] 论文主线至少覆盖两个数值来源家族后再写 generalized claim。

**Acceptance**

- [ ] 数据集扩展不是“再跑一个 benchmark”，而是扩展一种 visual numeric source family。
- [ ] 每个数据集进入 v2/v3 前都有明确 go/no-go：
  valid rate、source edit quality、`c_cf` reliability。

## 4. v1 - FinQA / Synthetic Table 最小闭环

### v1-1 FinQA On-Path Locator and Source Backend

**Spec**

v1 只做 FinQA rendered table，并可加 tiny synthetic table spike。目标不是扩大覆盖面，而是得到一个审稿人难以攻击的 clean path-specific cell。

主设置：

- dataset：FinQA rendered table。
- model：Qwen3-VL 8B FinQA curriculum arm，优先 `data/distill/poc/lora_8b_finqa_b2`。
- condition：present image/table。
- target：gold-program flippable table operand。
- visual source：对应 rendered table cell；必须能从 sample metadata/table text 定位到唯一 cell。
- counterfactual policy：优先沿用 `v_cf = 2 * v_true + 7`。
- v1 token policy：只保留 tokenizer 下 `v_true` 和 `v_cf` 都满足 single-token constraint 的样本。
- follow target：`c_cf = gold_program(v_true -> v_cf)`，且必须不同于 `c_true`。

### v1-2 Qwen3-VL Manual Greedy Decode Spike

**Spec**

v1 最大技术风险是 Qwen3-VL multimodal cache 是否能在 HuggingFace manual decode 中稳定使用。先做 1-3 条 FinQA sample spike，不进入规模实验。

必须验证：

1. manual greedy decode 与 `model.generate(... do_sample=False)` 输出一致或足够接近。
2. prefill 时喂 image/table，后续 token-by-token decode 只喂 token + `past_key_values`。
3. 在目标 token 前 clone/crop prefix cache 后，force 原始 `v_true` token 能 replay 出同一 final answer。
4. force `v_cf` token 后能继续同一 branch，不出现 shape/cache_position 错误。

**Steps**

- [ ] 在 `app/eval_distill/inplace_kv.py` 实现 `build_qwen_inputs(processor, image, question, prompt)`，沿用旧脚本 chat template。
- [ ] 实现 `manual_prefill(model, batch)`。
- [ ] 实现 `manual_next(model, token_id, past_key_values, attention_mask/cache_position)`。
- [ ] 实现 `clone_past(past_key_values)`：
  优先支持 `DynamicCache` clone/crop API；否则逐层 clone tensor。
- [ ] 实现 `force_continue(prefix_cache, forced_token_ids, max_new)`。
- [ ] 写 `scripts/spike_qwen3vl_inplace_one.py`：
  输出 base text、target token、replay text、injected text、cache diagnostics。
- [ ] 验证 replay control：
  从 `token_span` 强制原始 `v_true` 后，`base_ans` 和 replay answer 必须一致。

**Acceptance**

- [ ] 单样本 spike 能完成 base generation、target detection、prefix-cache clone、force `v_true` replay、force `v_cf` continuation。
- [ ] replay answer 与 base answer 一致；若 token 不完全一致，有明确、可复现的分叉日志。

### v1-6 FinQA Summary and Decision Gate

**Spec**

## 5. v1.5 - TabMWP Table Arithmetic Extension

### v1.5-1 TabMWP Locator and Source Backend

**Spec**

TabMWP 没有 FinQA gold program，因此采用 conservative parse-and-skip：

- 从 emitted CoT 解析最后一个能产生 base/gold answer 的 arithmetic expression。
- 只接受安全表达式：数字、括号、`+ - * /`、百分号 normalization、简单等号链。
- 选择一个 operand，使 `operand -> v_cf` 后 expression recompute 的 `c_cf` 与 `c_true` 不同。
- visual source 必须能定位到唯一 table cell 或题面数值。
- 找不到 source provenance 的 expression operand 只能用于 `text_only_inplace` appendix，不进完整 path-specific 主表。

复用：

- `app/distill/eval_common.py::relaxed_match`。
- 旧 TabMWP case loading/prompt 代码来自 `scripts/build_tabmwp_test.py`、`scripts/battery_n400.py`、`scripts/probe_n400.py`。

**Steps**

- [ ] 实现 `app/eval_distill/onpath_tabmwp.py`。
- [ ] 实现 safe arithmetic parser：
  用 Python `ast` whitelist 或手写 parser，禁止任意 `eval`。
- [ ] 支持常见模式：
  `a + b = c`、`a - b = c`、`a * b = c`、`a / b = c`、`(a+b)/c = d`、`Total = ...`。
- [ ] 实现 `app/eval_distill/source_tabmwp.py`：
  将 parsed operand 对齐到 table cell / problem field；多处出现则 skip。
- [ ] 实现 TabMWP counterfactual renderer。
- [ ] 添加 fixture tests：
  simple addition、subtraction、percentage、ambiguous expression、expression result not answer。
- [ ] dry-run 统计 `data/distill/tabmwp/test_cases_400.jsonl` 上 parse rate 和 expected valid rate。

**Acceptance**

- [ ] TabMWP locator 宁可 skip，也不接受无法重算 `c_cf` 的样本。
- [ ] 每个 valid TabMWP target 的 `c_cf` 来自 expression recompute，不是 injected operand 本身。
- [ ] 每个进入 path-specific 主表的 TabMWP target 都有 source provenance。

## 6. v2 - Non-Table Source Families and Multi-Token Support

- `v_cf` 可为不同长度 token span。

- 不复用 target 后原始 cache。

**Steps**

## 7. Statistics, Guardrails, and Paper-Readiness Gates

### Gate A: KV-Cache Replay

**Spec**

如果 Qwen3-VL cache branch 无法可靠 replay，不能继续规模跑。

**Steps**

- [ ] 若 replay_true answer match < 0.98，暂停 scale run。
- [ ] 检查 `cache_position`、attention mask、DynamicCache clone/crop、PEFT wrapper。
- [ ] 若只能通过 re-prefill 重建 prefix，标注为 `replay_prefill`，不得称为 C1-clean in-place。

**Acceptance**

- [ ] 只有 C1-clean spike 通过，才进入 v1 scale。

### Gate B: Valid n

**Spec**

v1/v1.5 的目标不是满 400，而是足够窄但可信的 valid subset。

**Steps**

- [ ] 若 valid n >= 150：进入主表。
- [ ] 若 `80 <= valid n < 150`：只进 appendix 或标注 underpowered。
- [ ] 若 valid n < 80：不作为 headline causal result，改为 spike evidence。

**Acceptance**

- [ ] 不用低 n 的 null 支撑强 claim。

### Gate C: Divergence

**Spec**

in-place 高 follow 是 thesis-changing，不是坏结果。

**Steps**

- [ ] 如果 in-place follow 明显高于 two-pass，冻结旧 "written chain not load-bearing under both paradigms" 表述。
- [ ] 新叙事改为：
  prompt-level re-query can destroy stream faithfulness; emitted chain may be load-bearing only in its native autoregressive context。
- [ ] 重新命名 paper title/subtitle，避免 `Reading, Not Reasoning` 过强。

**Acceptance**

- [ ] 结果解释先服从实验，不反向服务标题。

### Gate D: Review Pitfall Guardrails

**Spec**

每个 v1/v1.5/v2 结果包都必须通过方法透明度、统计、selection flow、control、claim boundary 和复现护栏。

**Steps**

- [ ] 每个 clean probe summary 输出：
  `n_raw -> n_base_correct -> n_cot_present -> n_locator_valid -> n_single_or_multi_token_valid -> n_scored`。
- [ ] 每个 path-specific summary 输出：
  `n_raw -> n_source_localizable -> n_source_editable -> n_cf_recomputable -> n_conflict_identifiable -> n_scored`。
- [ ] 每个 follow/snap/flip rate 带 95% CI。
- [ ] `corrupt_flip - control_flip` 带 paired bootstrap 或 exact randomization CI。
- [ ] 每个 path effect 带 paired bootstrap CI。
- [ ] multi-cell headline 结果标明 confirmatory vs diagnostic。
- [ ] skipped/failed locator 样本做 sensitivity bound。
- [ ] 每个主 cell 至少有一个 no-target local/offpath control；若没有，只能进 appendix。
- [ ] 每个 result snapshot 写明：
  `object audited = localized visual source and/or emitted written rationale`；
  `not audited = all other hidden-state / latent computation paths`。
- [ ] 没有视觉反事实前，把 `re-read` 降调为 "snap to recoverable source"。
- [ ] 如果没有非 Qwen 复现，Abstract/Discussion/Limitations 明确写 single model family。

**Acceptance**

- [ ] 任一结果进入主文前，都有方法透明度、CI、selection flow、control、claim-boundary 五项证据。
- [ ] 任一 visual grounding/source claim 进入主文前，都有 source localization、counterfactual rendering/editing、`c_cf` recompute、四格 conflict readout。

### Gate E: Paper Readiness Bundle

**Spec**

每个阶段不以“实验跑完”为完成，而以“实验 + 分析 + 文稿接入 + 合规材料”作为完成。

**Steps**

- [ ] v1 完成条件：
  FinQA clean text-path JSONL + path-specific JSONL + summary + CI + source/selection flow + same-target two-pass + artifact map。
- [ ] v1.5 完成条件：
  TabMWP clean text-path JSONL + path-specific source subset + summary + CI + no-target control + selection/source flow + cross-paradigm/path table。
- [ ] v2 完成条件：
  multi-token/chart-family/full controls/extra source family 至少一项完成，并更新 locator backend table、source backend table、skip-flow appendix、limitations。
- [ ] AAAI 当前稿可借用的最小修订包：
  P0-2 statistics、P0-3 selection bias、P0-4 cleaner control、P0-6 claim boundary、P1-1 field positioning、P1-5 Method title、双盲/复现合规。

**Acceptance**

- [ ] 每次推进一个版本，都能同步产出 paper-ready 证据，而不是只产出 raw experiment files。
